# Git 分支完整流程

## 1. 创建新分支
```bash
git checkout -b feature-branch
```

## 2. 开发阶段
```bash
# 查看当前分支
git branch

# 查看状态
git status

# 添加更改
git add .

# 提交更改
git commit -m "feat: 添加新功能"
```

## 3. 推送分支到远程（可选）
```bash
git push origin feature-branch
```

## 4. 合并到主分支
```bash
# 切换到主分支
git checkout main

# 拉取最新代码
git pull origin main

# 合并功能分支
git merge feature-branch

# 推送合并后的主分支
git push origin main
```

## 5. 删除分支
### 情况1：只创建了本地分支
```bash
git branch -d feature-branch
```

### 情况2：已推送到远程
```bash
# 删除本地分支
git branch -d feature-branch

# 删除远程分支
git push origin --delete feature-branch
```

## 6. 验证
```bash
# 查看本地分支
git branch

# 查看远程分支
git branch -r

# 查看所有分支
git branch -a
```

## 关键概念
- **本地分支**：只在本地存在，不影响远程仓库
- **远程分支**：推送到远程后存在，与本地分支关联
- **合并**：将代码更改集成到目标分支，不复制分支本身
- **删除**：清理已完成的功能分支，保持仓库整洁

## 最佳实践
- 功能完成后及时删除分支
- 使用有意义的分支命名（如 `feature/user-auth`）
- 合并前确保主分支是最新的
- 优先使用 Pull Request 进行代码审查
- 统一使用 `main` 作为主分支名称
